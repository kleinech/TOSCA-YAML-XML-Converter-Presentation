### Namespaces TOSCA XML Namespaces ```yaml TOSCA: http://docs.oasis-open.org/tosca/ns/2011/12 http://www.opentosca.org/winery/extensions/tosca/2013/02/12 ``` TOSCA YAML Namespaces ```yaml TOSCA YAML: http://docs.oasis-open.org/tosca/ns/simple/yaml/1.1 Default: http://www.eclipse.org/winery/ns/simple/yaml/1.1/default ``` User defined Namespaces ```yaml http://www.kleinech.uni-stuttgart.de/tosca/ns/ ```
#### OpenTOSCA Namespace conventions Namespace prefix ```yaml http://www.opentosca.org/ ``` followed by TOSCA component type ```yaml e.g. servicetemplates, nodetypes, ... ```
### Description and Documentation
#### Description Single line descriptions ```yaml description: string ``` Multi line descriptions ```yaml description: > this is the first line of the description this is appended to the first line of the description this is the second line of the description ``` See [YAML multi line strings](https://stackoverflow.com/a/21699210/8235252) for more different ways to write multi line descriptions.

Attribute definition

  • Defines a named and typed value and is used to expose the actual state of some property of a TOSCA entity.
  • TOSCA orchestrators automatically create Attribute definitions for every Property Definition


Grammar
AttributeDefinitionName:
    type: DataType
    description: Description
    default: AttributeDefault
    status: StatusValue

Attribute default

MUST be derived or calculated from other Attributes or Operation output values

Example
AttributeName:
    type: string
    default: { get_operation_output: [ SELF, iface, op, name ] }
